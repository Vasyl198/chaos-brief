# Chaos Brief — 60 second demo plan

## 0–8s — problem

Open the empty app and say:

> Work requests often look clear, but a linear plan hides the assumptions that make it fail later.

## 8–20s — input and output

Click **Use a realistic example**, then **Generate decision brief**.

> Chaos Brief turns a vague request into a decision brief. It makes the next decision, concrete acceptance criteria and evidence gaps visible before more work is committed.

## 20–40s — scenario interaction

Click **Changing conditions**, then **Mature-alternative check**.

> Instead of assuming one happy path, it checks six paths: an ordinary path, bad data, changed conditions, a workaround, and whether an existing product is the cheaper answer.

## 40–52s — output

Click **Copy as Markdown**.

> The result is a portable brief: it can be copied into a task, shared with a client, or used as the agreement for the next review.

## 52–60s — Codex and GPT-5.6 disclosure

> I developed this project in Codex with GPT-5.6. Together they helped turn the idea into the scenario-first workflow, implement and validate the JavaScript, and test the live user flow in a browser. The public app itself uses a transparent deterministic engine rather than an OpenAI API call.

Do not state that the in-browser scenario engine calls an OpenAI model: this prototype deliberately uses a transparent, deterministic engine.
